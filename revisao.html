Como Publicar Projetos Web

Aprenda a usar GitHub Pages, Vercel, Netlify e criar um README eficiente

GitHub Pages Vercel Netlify README

1. Publicar no GitHub Pages

GitHub Pages é uma forma simples de hospedar sites estáticos diretamente de um repositório GitHub.

  1. Crie uma conta no GitHub.
  2. Crie um repositório público e faça upload do seu site (index.html, CSS, JS).
  3. No repositório, vá em Settings → Pages e escolha o branch main.
  4. Salve e aguarde alguns minutos. O site estará disponível em https://seu-usuario.github.io/nome-do-repositorio/.

2. Publicar no Vercel

Vercel é uma plataforma de deploy rápido para sites estáticos e aplicações JavaScript.

  1. Crie uma conta em Vercel.
  2. Conecte sua conta do GitHub e escolha o repositório do projeto.
  3. Configure as opções de build (ou use padrão) e clique em Deploy.
  4. O site ficará disponível em um link do tipo https://nome-do-projeto.vercel.app.

3. Publicar no Netlify

Netlify é outra plataforma que facilita o deploy de sites estáticos com integração contínua.

  1. Crie uma conta em Netlify.
  2. Clique em New site from Git e conecte o repositório do GitHub.
  3. Configure as pastas de build se necessário e clique em Deploy site.
  4. O site será publicado com uma URL do tipo https://nome-do-site.netlify.app.

4. Criando um README

O README é o cartão de apresentação do seu projeto no GitHub:

  • Explique o que o projeto faz.
  • Mostre como instalar, configurar ou usar.
  • Inclua imagens ou links se necessário.
  • Use formatação Markdown para títulos, listas e código.

Exemplo de README básico:

# Meu Projeto
Descrição curta do projeto.

## Como usar
1. Clonar o repositório
2. Abrir o arquivo index.html
